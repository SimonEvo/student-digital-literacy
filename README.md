# 学生电子扫盲 (Student Digital Literacy Guide)

> **状态：想法记录（idea record），非架构决策，无 ADR。** 这是一个轻量、易反悔的静态页项目。本文档记录构思，供日后开工时参照。相关术语见 EPQ 家教系统仓库根 `CONTEXT.md` 的「学生电子扫盲」段。

## 是什么

一个**面向学生、纯静态的自助指南**，讲清导师反复重复解释的 "book keeping" 操作类问题 —— 尤其是给年纪小的学生：表格怎么填、在线文档怎么用才不丢数据、电脑基本操作怎么做。

导师目标：**甩掉这部分重复劳动**。以后不再 1:1 口头讲，课上展示这一页，学生日后自己查。

## 为什么是静态页，而不是学生问 AI

学生侧实时 AI（让学生自己问）在**成本结构**上对单人导师不成立，不是想法不好：
- 公司的在线填表系统后端**不在导师手里**，碰不了。
- 学生侧实时 AI → token 随学生数 / 提问量线性烧钱，还要服务器扩容 → 单人扛不住。

而 90% 的问题是**同一批重复问题**，一份常青的静态内容就够。剩下 10% 新奇 / 个性化的，导师本来就得亲自处理，没损失。

## 形态

- **独立静态 HTML 页**，Nginx 托管 —— 复用 EPQ 系统里 `gantt-pro.html` 的模式（见该仓库 `docs/adr/0001-gantt-editor-as-standalone-html.md`）。
- 文字 + 截图 + 短 GIF / 视频。
- **无需登录**（不含敏感数据）、**无实时 AI、无 per-query 成本、无扩容**。
- **不接触公司填表系统**，只"讲怎么做"。

## Mobile-first（硬约束）

学生**手机扫码**进来，所以从第一行代码就必须移动优先：窄屏布局、大字大按钮、点击区域够大。**不是**把桌面页缩小。

## 访问路径 —— 一等问题，不是事后补丁

**很多学生连"把网页存书签 / 再次找到它"都做不到。** 所以触达方式必须极简、可反复分发：

- **主打二维码（QR）**：课上屏幕 / 材料上放 QR，学生手机扫一下即进历史 / 加主屏。比"记网址存书签"门槛低一个量级。（还刚好谐音"扫盲/扫码"。）
- **兜底**：一个短好记的路径，如 `<域名>/help`。
- QR 可**印在发给学生的纸质 / PDF 材料**上，反复出现 → 反复触达。

## 内容清单（四类全要）

- **A. 表格填写** —— 每类表一节：EPQ tables 1 / 2 / 4 / 5 / 6 / 7 / 11 分别怎么填、常见错误。
- **B. 在线文档 / 防丢数据** —— 怎么存、自动保存、版本历史、为什么只存本地会丢。
- **C. 电脑基本操作** —— 复制粘贴、截图、找文件、上传附件等。
- **D. EPQ 流程扫盲** —— 这个 EPQ 项目大概是什么、有哪些步骤、学生现在走到哪。（与"家长可视时间线"想法有轻微呼应，保留。）

具体 FAQ 措辞**留到开工时**定 —— 导师那时挑最常被重复问的真实问题填进去。

## AI 的角色 —— 只帮写，不实时答

AI 只用来**一次性帮导师起草指南内容**（一次性、便宜，甚至用网页版 AI 就行），**绝不**实时回答学生。

## 与"学生知识库"的区别

本项目内容是**通用 / 共享的**（对所有学生一样）、常青，**不分学生、不分学期**。
而「学生知识库」（EPQ 系统里的另一个项目）是**导师私有、一人一份**的。两者不要混。
