# 0004. 点击式互动、学生零输入；多文件静态站点

- 状态：已接受（Accepted）
- 日期：2026-07-07
- 决策者：导师（单人）

## 背景与问题

互动性强已定为硬要求（目标用户年纪小，长文读不进，学生是来「照着做」的，不是来「读」的）。问题：互动采取什么形态？

关键判断：**任何要学生主动输入 / 练习的形式都会失败** —— 很多学生懒得做练习。「先练一遍再去填真表」的练习沙盒模式，看起来教学效果好，实际没人练。

同时，互动内容变多后，ADR [0001](0001-static-html-guide-not-realtime-ai.md) 沿用的「单 HTML 文件」（gantt-pro.html 模式）会越来越胖，不再合适。

## 决策驱动因素

- 学生懒 → 互动门槛必须趋近于零：点一下就有反馈，不打字、不填东西。
- 互动必须纯前端，守住 [0001](0001-static-html-guide-not-realtime-ai.md) 的零后端 / 零运维地基。
- 互动内容体量 → 单文件不可维护。
- 无学生输入 → 顺带消除数据留存 / 隐私问题，继续无需登录。

## 备选方案

互动形态：

1. **纯点击式**：学生只点 / 滑，零输入 —— 分步向导、症状选择器、点击播放 GIF、折叠分节等。
2. **练习沙盒**：页面内仿真表格，学生先练一遍再去填真表。
3. **自测小测验**：每节 2-3 题，答题得反馈。

文件结构：

- (a) 单 HTML 文件（gantt-pro.html 模式）；(b) **多文件静态站点**（多页面 / 拆分 CSS、JS），仍 Nginx 托管。

## 决策结果

- 互动形态：**方案 1，纯点击式，学生零输入**。
- 文件结构：**(b) 多文件静态站点**，Nginx 托管不变，仍纯静态、零后端。

### 后果

好处：

- 互动门槛趋近于零，贴合「学生懒」的现实，不指望学生主动性。
- 零输入 → 零数据留存顾虑，无需登录的前提更稳。
- 多文件 → 每类表 / 每个主题可独立成页，内容维护按页改，不再挤一个大文件。

代价 / 取舍：

- 放弃练习类互动的教学效果 —— 但没人练的练习教学效果为零，无净损失。
- 多文件比单文件部署稍复杂（一个目录 vs 一个文件），仍在「拷文件到 Nginx」量级，可接受。
- 放宽了 [0001](0001-static-html-guide-not-realtime-ai.md) 的「单 HTML 文件」细节（0001 的核心决策 —— 静态而非实时 AI —— 不变）。

## 方案优劣对比

### 练习沙盒（未选）

- 好处：「练一遍再去做」教学效果理论上最好。
- 坏处：学生懒得练 → 实际使用率趋零；且每类表要仿一份 UI，公司表格改版还得跟着改，成本最高、收益落空。

### 自测小测验（未选）

- 好处：即时反馈，轻量。
- 坏处：仍要学生主动答题，同样撞上「懒」这堵墙；与「照着做」的场景（手头正卡在某一步）也不匹配。

### 单 HTML 文件（未选）

- 好处：部署最简单，已验证模式。
- 坏处：互动内容多 → 文件膨胀，改一处要在大文件里翻找，可维护性差。
